GIT

Для сучасних бізнес-аналітиків набуття технічних навичок стає важливим елементом конкурентоспроможності. Системи контролю версій (скорочено VCS, від англ. Version Control Systems) значно спрощують роботу з файлами та забезпечують відстеження усіх змін. За допомогою VCS можна уникнути втрати даних, просто відкотитися до попередніх версій або виправити помилки, що були допущені в коді або документації.

Система контролю версій — це інструмент, що дозволяє створювати та координувати роботу над одним проектом, відстежуючи всі зміни, що вносяться командою або окремими користувачами.

Git та SVN

Git і SVN (система Subversion) поширені в різних сферах. Завдяки своїм особливостям вони мають різні сценарії застосування:

Git

Децентралізована система контролю версій. Дані зберігаються на кожному комп’ютері користувачів.
Переваги: швидкість, можливість роботи офлайн, одночасна підтримка кількох версій продукта.
Недоліки: складність для новачків.
Типові сфери використання: стартапи, open-source проекти, проекти з великою кількістю розгалужень.

SVN

Централізована система, де всі зміни зберігаються на сервері.
Переваги: простота використання, підходить для невеликих команд.
Недоліки: залежність від сервера: якщо сервер з репозиторієм стане недоступним, відслідковувати зміни, внесені іншими користувачами, стане неможливо.
Типові сфери використання: корпоративні середовища, де необхідний строгий контроль доступу, та проекти, що побудовані на застарілому стеку технологій та потребують підтримки.

Переваги GIT над іншими системами:

Головна перевага Git – в тому, що він дуже швидкий і прозорий. Він зручний для нелінійної розробки і ефективний як для невеликих проектів, так і для великих систем з тисячами учасників.

На відміну від Perforce, CVS та інших, Git зберігає знімки репозиторіїв, а не списки змін в файлах, і внаслідок цього працює набагато швидше.

Git – розподілена система. Якщо сервер з віддаленим репозиторієм вийде з ладу, можна відновити код з локальної копії. Якщо локальна копія постраждає, можна завантажити код із сервера за кілька хвилин.

10 основних команд у GIT

git init - використовується для ініціалізації нового репозиторію.

git add - використовується для додавання файлів до індексу Git перед їх фіксацією. Це дозволяє GITу відстежувати зміни у цих файлах щоб додати їх у коміт

git commit - використовується для фіксації змін у репозиторії.

git branch <ім’я гілки> - команда використовується для створення нових гілок.

git checkout <ім’я гілки> - Створення нової гілки та перемикання на неї

git merge - використовується для злиття змін з однієї гілки в іншу.

git revert - якщо вам потрібно скасувати зміни, які були вже опубліковані або перебувають у загальному доступі для інших членів команди.

git reset - у випадку, коли вам потрібно внести зміни в локальну історію комітів.

git rebase - використовується для перебазування комітів на іншу гілку або базовий коміт.

git push origin main -команда, щоб відправити свої зміни на віддалений репозиторій

Сертифікат

LINUX

GNU/Linux — це вільна, Unix-подібна операційна система, яка складається з ядро Linux, розробленого Лінусом Торвальдсом, та системних програм проекту GNU.

Це колективний проєкт з відкритим кодом, що існує у вигляді багатьох "дистрибутивів", які об'єднують ядро з іншими програмами для створення повноцінних операційних систем. GNU/Linux є фундаментальною частиною багатьох пристроїв, включаючи сервери, суперкомп'ютери та смартфони, і пропонує користувачам велику свободу вибору та гнучкість.

Найбільш популярні дистрибутиви Linux

Ubuntu — найкращий вибір для новачків

Підходить для: початківців, офісної роботи, навчання
Середовище робочого столу: GNOME
Оновлення: кожні 6 місяців + LTS-підтримка 5 років
Переваги: величезна спільнота, простий інтерфейс

Linux Mint — стабільність та зручність

Підходить для: домашнього використання, переходу з Windows XP/7
Середовище робочого столу: Cinnamon, MATE, Xfce
Переваги: легкий інтерфейс, стабільність
Мінуси: не найсвіжіші оновлення

Fedora — технології майбутнього

Підходить для: розробників, DevOps
Середа: GNOME
Переваги: свіжі пакети, SELinux, стабільна база
Мінуси: менше термін підтримки, ніж у Ubuntu LTS

Debian – перевірена класика

Підходить для: серверів, досвідчених користувачів
Середа: будь-яка (на вибір)
Переваги: стабільність, висока надійність
Мінуси: складніше в установці та налаштуванні

Arch Linux — свобода та контроль

Підходить для: просунутих користувачів
Середа: вибирається вручну
Переваги: свіжі пакети, гнучкість
Мінуси: складна установка, немає GUI за замовчуванням
